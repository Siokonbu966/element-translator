# Element Translator

Webページ上のテキスト要素（パラグラフなど）を選択・クリックして、ローカルLLMを用いて日本語に翻訳するブラウザ拡張機能です。外部の翻訳APIにデータを送信することなく、セキュアかつ完全にプライベートな環境で翻訳を実行できます。

## ✨ 特徴 (Features)

- **ローカルLLMによる完全なプライバシー**: [LM Studio](https://lmstudio.ai/) のローカルサーバー機能を利用して翻訳処理を行います。
- **自動モデル連携**: LM Studioでロードされているモデル一覧を自動取得し、ポップアップ画面から簡単に切り替え可能です。
- **翻訳履歴の保存**: 過去の翻訳履歴（対象URL、日時、使用モデル、原文、訳文）を拡張機能のポップアップからいつでも確認できます。

## 🚀 必要な環境 (Prerequisites)

拡張機能を使用するには、以下の準備が必要です。

- **LM Studio**: インストール済みで、日本語対応のLLMモデルがロードされていること。
- **Local Serverの起動**: LM Studio内でローカルAPIサーバー（デフォルト: `http://127.0.0.1:1234`）が稼働していること。

## 🛠️ ビルドとインストール (Build & Install)

**開発・ビルド要件**

- Node.js >= 24
- pnpm >= 10
- (推奨) [Nix](https://nixos.org/) - `flake.nix` をサポートしています。

### ビルド手順

Nix環境がある場合は、以下のコマンドで再現性のある開発環境を立ち上げてビルドできます。

```sh
# 1. 開発環境のセットアップ (Node.js 24 と pnpm が準備されます)
nix develop

# 2. 依存関係のインストール
pnpm install

# 3. ビルドの実行
pnpm build
```

※ Nixを使用しない場合は、直接ローカルの Node.js / pnpm を使用して `pnpm install` と `pnpm build` を実行してください。

ビルドが成功すると、プロジェクト内に `dist/` フォルダが生成されます。

### ブラウザへの導入 (Chrome / Edge などの Chromium 系)

1. ブラウザのアドレスバーに `chrome://extensions/` (Edgeの場合は `edge://extensions/`) を入力して開きます。
2. 画面右上にある **「デベロッパー モード」** をオンにします。
3. **「パッケージ化されていない拡張機能を読み込む」** をクリックし、ビルドで生成された `dist` ディレクトリを選択します。

## 📖 使い方 (Usage)

1. **拡張機能の初期設定**
   - ブラウザのツールバーから「Element Translator」のアイコンをクリックしてポップアップを開きます。
   - `Refresh models` ボタンをクリックして、LM Studioで現在起動しているモデルを認識させます。
   - ドロップダウンから使用したいモデルを選択します。
2. **翻訳する**
   - 任意のWebページを開き、翻訳したい文章（パラグラフ）の要素をクリック・選択します。
   - バックグラウンドでLM Studioへの通信が行われ、翻訳結果が保存されます。
3. **結果の確認**
   - 再度ポップアップを開くと、画面下部の「History」に原文と翻訳結果が表示されます。
